A Small Introduction to OCaml

This page is ordinary HTML. Each runnable example below starts as an <oxcaml> tag and is turned into a typecheck-and-run editor by one script at the bottom of the page.

Expressions and Values

OCaml programs are built from expressions. A let binding names the result of an expression, and the compiler infers the type. This first editor is in utop mode, so each phrase ending in ;; prints the value and inferred type as you edit.

let language = "OCaml";; let year = 1996 + 1;; let message = Printf.sprintf "%s has had type inference since %d." language year;;

Functions

Functions are values too. This function has no type annotation, but OCaml still knows that square takes an integer and returns an integer.

let square x = x * x let () = List.iter (fun x -> Printf.printf "%d squared is %d\n" x (square x)) [1; 2; 3; 4; 5]

Lists and Recursion

Lists are immutable. Recursive functions consume them one case at a time: the empty list, or a head element followed by the rest of the list.

let rec sum numbers = match numbers with | [] -> 0 | first :: rest -> first + sum rest let () = Printf.printf "sum = %d\n" (sum [3; 1; 4; 1; 5; 9])

Variants and Pattern Matching

Variants let you define a small language of cases. Pattern matching makes the compiler check that you handled the cases you said exist.

type shape = | Circle of float | Rectangle of float * float let area shape = match shape with | Circle radius -> Float.pi *. radius *. radius | Rectangle (width, height) -> width *. height let () = Printf.printf "area = %.2f\n" (area (Rectangle (3.0, 4.0)))

Types Catch Mistakes Early

The editor also shows compiler errors as you type. The example below tries to put an integer where a string was promised. Fix it by changing 42 to a string literal.

let message : string = 42 let () = print_endline message
